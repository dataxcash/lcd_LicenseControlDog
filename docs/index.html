■Getting started with LCD

1
Pick the right binary
Go to the release/ directory in this repository (or the GitHub Release page) and download the LCD binary that matches your target platform.
release/linux-amd64/lcd   # platform example only
2
Place it next to your application
Deploy LCD alongside your service or application, following your internal deployment guidelines. The exact invocation flags and configuration are defined by your team’s operations playbook.
3
Connect to LCC
In a typical setup, LCD communicates with the License Control Center running in your environment. Configuration of addresses, credentials, and trust anchors is handled outside of this public documentation.
4
Integrate from your code
Your application calls into LCD through a narrow interface (for example, a local HTTP or IPC endpoint) to check whether a given feature, tenant, or operation is licensed.

Important: this site only covers the conceptual flow. Exact command-line flags, configuration formats, and integration snippets should be kept in your internal engineering documentation, not in this public repo.

■Component Roles

License Control Center (LCC)

The central local hub. Applications connect here (via SDK) to check permissions. LCC aggregates data and manages the local license pool.

LMF / upstream

The global authority. LCC connects upstream to LMF to fetch licenses, report usage, and synchronize entitlements.

Apps → connect to LCC (via SDK) LCD → reports device fingerprint to LCC LCC → syncs with LMF

In this architecture, LCD acts as a hardware probe and environment monitor, ensuring LCC is bound to the correct physical device, while applications interact solely with LCC.
